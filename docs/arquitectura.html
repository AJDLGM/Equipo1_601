Sistema de Identidad Digital

Arquitectura, diseño y componentes del sistema de gestión de usuarios con firma criptográfica

Arquitectura General

Capas del Sistema

🖥️

Capa 1 — Presentación (Cliente Tkinter)

Interfaz gráfica de escritorio que corre en la PC del usuario final

  • main.pyPunto de entrada. Verifica conexión al servidor antes de mostrar la UI.
  • ui/login_window.pyPantalla principal: login, registro, firma y verificación de archivos/mensajes.
  • ui/admin_panel.pyPanel exclusivo para administradores: gestión de usuarios, logs y revocaciones.
🔌

Capa 2 — Comunicación HTTP

Puente entre el cliente y el servidor REST

  • client/api_client.pyEnvía peticiones HTTP con token Bearer. Guarda y reutiliza la URL del servidor.
  • config/server_config.pyLee y guarda la URL del servidor en server_config.json para persistirla entre sesiones.
  • config/paths.pyDefine rutas de directorios de datos (DB, claves, certificados).
⚙️

Capa 3 — API REST (Servidor Flask)

Corre en el servidor; expone endpoints para autenticación, usuarios, claves y logs

  • server/app.pyDefine todos los endpoints Flask. Maneja sesiones en memoria con tokens hex.
  • auth/auth.pyLógica de registro y login. Hasheo PBKDF2-HMAC-SHA256 con salt aleatorio.
  • auth/permissions.pyTabla de permisos por rol. Función has_permission(role, perm).
  • crypto/keys.pyGenera par de claves RSA 2048 bits por usuario y las guarda en la BD.
  • crypto/certificate.pyCrea certificado JSON (emisión, expiración 1 año, firma SHA-256) por usuario.
  • crypto/signature.pyFirma y verifica archivos/mensajes usando la clave privada RSA del usuario.
  • db/admin_queries.pyConsultas admin: listar, aprobar, revocar y desactivar usuarios.
  • db/logs.pyRegistra cada acción relevante con usuario, descripción y timestamp.
🗄️

Capa 4 — Persistencia (SQLite)

Base de datos local embebida en el servidor

  • db/database.pyCrea las tablas, gestiona la conexión y migra claves de archivos a la BD si existían.

Mecanismos de Seguridad

🔐

Contraseñas

PBKDF2-HMAC-SHA256 con salt aleatorio de 16 bytes y 200,000 iteraciones. Nunca se almacena la contraseña en texto plano.

🎟️

Sesiones

Token hex de 64 caracteres generado con secrets.token_hex(32) al hacer login. Se almacena en memoria del servidor y se envía como Bearer Token en cada petición.

🔑

Criptografía Asimétrica

Cada usuario recibe un par de claves RSA 2048 bits al ser aprobado. La clave privada se guarda en la BD del servidor y puede descargarse solo por el propio usuario o un admin.

📜

Certificados de Identidad

Certificado JSON con usuario, fecha de emisión, expiración (1 año) y firma SHA-256 del contenido. Puede ser revocado y aparece en la CRL.

✍️

Firma Digital

El cliente puede firmar archivos y mensajes localmente con su clave privada RSA. Cualquier usuario puede verificar una firma con la clave pública correspondiente.

📋

Auditoría

Cada acción relevante queda registrada en la tabla logs: logins exitosos, fallidos, registros, aprobaciones y revocaciones, con timestamp.

Despliegue

🖥️ Servidor Local

  • Ejecutar server/run_server.bat en Windows
  • Corre en http://0.0.0.0:5000
  • Accesible desde red local por IP del equipo
  • Puerto configurable con variable de entorno PORT

☁️ Nube (Railway)

  • Deploy usando wsgi.py como entrypoint
  • Railway inyecta la variable PORT automáticamente
  • El cliente solo necesita cambiar la URL configurada
  • SQLite persiste en el filesystem del contenedor

📦 Cliente Distribuible

  • Compilar con build.bat usando PyInstaller
  • Genera dist\SistemaIdentidad\SistemaIdentidad.exe
  • Distribuir toda la carpeta, no solo el .exe
  • No requiere Python instalado en el cliente
